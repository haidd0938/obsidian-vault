### ⚠️ 陷阱：用 re.sub() 替换 SCENES 会吃掉 `\n`

```python
# ❌ 错误做法 — 这个会把 \n 变成真换行，导致 SyntaxError
import re
content = re.sub(r'SCENES = \[.*?\]', new_scenes, content, flags=re.DOTALL)

# ✅ 正确做法 — 用 patch() 工具逐块替换
# patch(old_string=..., new_string=..., path=script.py)

# ✅ 如果必须用 Python 操作，确保替换字符串用 \\n 而不仅是 \n
new_scenes = 'SCENES = [{"text": "第一行\\n第二行", ...}]'  # \\n 在字符串里才是 \n
```

### ⚠️ BGM失效率：原因从网络下载扩展到tmp清理

**新发现的陷阱：** BGM文件 `/tmp/bgm_snooker.mp3` 在前一次运行时尚在，但后续cron/手动运行时已不存在。原因是系统重启或tmp清理删除了 `/tmp/` 下的文件。

**解决方案：**
1. BGM应存到持久化路径（如 `~/Desktop/_鑫球汇/bgm_snooker.mp3`），脚本同时检查 `/tmp/` 和持久化路径
2. 脚本 `get_bgm()` 函数应加优先检查顺序：持久化路径 → `/tmp/` → fallback None
3. 如果BGM全部缺失，直接跳过（不要用粉红噪音替代，用户不接受）
4. **即使持久化路径也可能不存在** — 需要手动放置BGM文件。如果网络不可用，`ffmpeg -f lavfi -i "sine=frequency=55:duration=78" -c:a aac` 生成的aac文件是无效的（FFmpeg 8.1 验证失败）。**无BGM的配音视频（14MB/78s）是可接受的交付，不要浪费时间尝试生成BGM。**
