```

   (P2)          (P4)         (P1)         (P3)       (P5)
```

### 3.3 复杂度评估算法

| 评分维度 | 条件 | 加分 |
|:---|:---|:---:|
| 文本长度 | >100字 / >500字 / >1000字 | +1 / +2 / +1 |
| 分析深度 | 含"分析/解释/为什么/比较" | +2 |
| 技术难度 | 含"代码/算法/架构/优化" | +2 |
| 创意需求 | 含"创意/写作/小说/诗歌" | +2 |

**路由阈值：**
- 1-3分：NAS层（1.5B模型，极速响应）

- 7-10分：云端层（DeepSeek，深度能力）

## 四、详细实施方案

### 4.1 笔记本（智能网关）部署

**硬件要求：** 能长期开机，稳定运行Python服务

**部署步骤：**

```bash
# 1. 安装依赖
pip install requests flask

# 2. 创建网关服务
mkdir -p ~/llm-gateway && cd ~/llm-gateway

# 3. 保存智能网关代码（smart_gateway_v2.py）
# [代码见上文完整实现]

# 4. 创建启动脚本
cat > start.sh << 'EOF'
#!/bin/bash
cd ~/llm-gateway
export GATEWAY_HOST=0.0.0.0
export GATEWAY_PORT=8080
python3 smart_gateway_v2.py
EOF
chmod +x start.sh

# 5. 设置开机自启（systemd）
sudo tee /etc/systemd/system/llm-gateway.service << EOF
[Unit]
Description=LLM Smart Gateway
After=network.target

[Service]
Type=simple
User=$USER
WorkingDirectory=/home/$USER/llm-gateway
ExecStart=/home/$USER/llm-gateway/start.sh
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable llm-gateway
sudo systemctl start llm-gateway
```

**配置清单：**
- 监听地址：`0.0.0.0:8080`
- API格式：OpenAI兼容 `/v1/chat/completions`
- 健康检查：每15秒轮询所有节点

### 4.3 家里NAS部署

**硬件要求：** 任何x86/ARM NAS，4G+内存即可

**Docker部署（推荐）：**

```yaml
# docker-compose.yml
version: '3.8'

services:
  ollama-nas:
    image: ollama/ollama:latest
    container_name: ollama-nas
    ports:
      - "11434:11434"
    volumes:
      - /volume1/ai/models:/root/.ollama
    environment:
      - OLLAMA_KEEP_ALIVE=1h      # 不常驻，节省资源
      - OLLAMA_NUM_PARALLEL=2      # 低并发
      - OLLAMA_MAX_LOADED_MODELS=1 # 单模型
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 4G
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:11434/api/tags"]
      interval: 30s
      timeout: 10s
      retries: 3
```

**部署命令：**
```bash
mkdir -p /volume1/ai/models
docker-compose up -d

# 拉取轻量模型
docker exec -it ollama-nas ollama pull qwen2.5:1.5b
docker exec -it ollama-nas ollama pull phi3:mini
```

### 4.4 公司主力电脑部署

**特点：** 可能断电，需要自动恢复

**部署步骤：**

```bash
# 1. 安装Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 2. 拉取办公模型
ollama pull qwen2.5:3b

# 3. 创建systemd服务（带自动重启）
sudo tee /etc/systemd/system/ollama-office.service << EOF
[Unit]
Description=Ollama Office Node
After=network.target

[Service]
Type=simple
User=$USER
Environment="OLLAMA_HOST=0.0.0.0:11434"
Environment="OLLAMA_KEEP_ALIVE=8h"
ExecStart=/usr/local/bin/ollama serve
Restart=always
RestartSec=30
StartLimitInterval=300
StartLimitBurst=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable ollama-office
sudo systemctl start ollama-office
```

### 5.2 跨网络方案（公司-家里）

**方案A：WireGuard VPN（推荐）**
```bash
# 家里NAS作为VPN服务器
docker run -d \
  --name=wireguard \
  --cap-add=NET_ADMIN \
  -e PUID=1000 -e PGID=1000 \
  -p 51820:51820/udp \
  -v /volume1/wireguard:/config \
  linuxserver/wireguard
```

**方案B：FRP内网穿透**
```ini
# frpc.ini（公司NAS）
[common]
server_addr = 家里公网IP/域名
server_port = 7000

[ollama-nas]
type = tcp
local_ip = 127.0.0.1
local_port = 11434
remote_port = 11435  # 映射到家里11435端口
```
