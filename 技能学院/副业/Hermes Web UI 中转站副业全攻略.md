## 一、底层认知：你在卖什么

### 核心逻辑

你手上有一个很值钱的东西：**Hermes Web UI**。

这个 Web UI 自带一个侧边栏菜单叫「中转站」，点进去是 apikey.fun 的注册页，**带你的返佣链接**。也就是说：

> **每一个用你部署的 Web UI 的人，都天然看得到这个菜单。** 这不是你主动推销，是界面自带的。

你不需要群发广告、不需要发朋友圈、不用当微商。**你的变现渠道就写在代码里。**

### 三条利润路径

```
路径 A：被动返佣（最懒的模式）
你 → 注册 apikey.fun → 获取返佣链接
         ↓
别人用你的 Web UI → 看到菜单 → 点进去注册 → 你拿返佣
         ↓
你自己都不用说话，菜单替你说话

路径 B：差价转卖（利润最高的模式）
你 → 在 apikey.fun 充值（官方 1/10 价格）
         ↓
创建子 Key → 按自己的定价卖给别人（比如原价 30%）
         ↓
一个月 10 个客户 = 月入几千

路径 C：打包服务（适合小白客户）
你 → 收 ￥198 帮对方部署一套 Hermes Web UI
         ↓
附送一个「已充值的中转 Key」
         ↓
后续对方继续充值 → 你从 apikey.fun 拿持续返佣
```

## 二、apikey.fun 是什么

一个**聚合 AI API 中转平台**，背后基于开源项目 [One API](https://github.com/songquanpeng/one-api) 搭建。

### 它解决了什么问题

官方 AI 模型（Claude、GPT、Gemini 等）在中国大陆的使用痛点：

| 痛点 | apikey.fun 的解决方案 |
|------|----------------------|
| 需要海外信用卡 | 支持微信/支付宝充值 |
| 每个模型一个 API | 统一 API Key，一个地址调用所有模型 |
| 价格贵 | 批量采购 + 渠道价，价格是官方的 1/10 |
| 被封号没法处理 | 充多少用多少，不怕封 |

### 主要功能

- ✅ 微信/支付宝充值（人民币）
- ✅ 模型自动管理
- ✅ 用量统计
- ✅ TOTP 二步验证
- ✅ 渠道管理
- ✅ **返佣系统（已验证开启）**
- ❌ 邀请码体系（关闭）
- ❌ 优惠码体系（关闭）

### 定价对比（以 Claude 为例）

| 模型 | 官方价格 (¥/1M tokens) | 平台价格 (¥/1M tokens) | 折扣 |
|------|----------------------|----------------------|------|
| Opus 4-7 (输入) | 35.00 | 3.50 | **90% off** |
| Opus 4-7 (输出) | 175.00 | 17.50 | **90% off** |
| Sonnet 4-6 (输入) | 21.00 | 2.10 | **90% off** |
| Haiku 3.5 (输入) | 3.00 | 0.30 | **90% off** |
| GPT-4o | 官方价 | 约官方价 15% | ~85% off |

> 数据来源：apikey.fun 公开定价页。并非所有模型都这个折扣，但 Claude 系列是，Claude 也是目前**需求最大、溢价空间最高的模型**。

## 四、三种模式的详细拆解

### 模式 A：被动返佣

**流程图：**
```
你 → 注册 apikey.fun → 获得返佣链接 (aff=LIBAPI)
         ↓
这个链接被硬编码在 Hermes Web UI 的侧边栏
         ↓
别人打开你的 Web UI → 看到「中转站」菜单 → 点进去
         ↓
对方注册并充值 → 你拿返佣
         ↓
你什么都不用做
```

**优点：** 完全被动，零维护
**缺点：** 返佣比例未知（需要登录后台才能看到），可能不高
**收入预估：** 取决于你的 Web UI 有多少人用
**关键问题：** 返佣比例多少？这个是注册后才能看到的

### 模式 B：差价转卖（核心利润模式）

**流程图：**
```
你 → 在 apikey.fun 充值 ¥200
         ↓
平台给你一个主 Key
         ↓
你在后台创建「子 Key」 → 设置不同的额度
         ↓
| 子 Key A：￥49/月 → 卖给小白用户
| 子 Key B：￥99/月 → 加量版
| 子 Key C：￥500 → 企业定制包
         ↓
客户付给你钱，你用子 Key 给他们用
         ↓
你的利润 = 卖价 - 平台成本
```

**利润测算：**
```
假设你充 ¥200，成本价买 Claude Opus：
- 平台输入成本：¥3.50 / 1M tokens
- 你卖 ¥10 / 1M tokens
- 利润率：185%

一个普通用户一个月用 3M tokens（重度用户）：
- 你的收入：￥30
- 你的成本：￥10.50
- 利润：￥19.50
- 10 个客户：￥195 / 月
- 50 个客户：￥975 / 月
```

**关键问题：** apikey.fun 是否支持创建子 Key？这需要登录后台确认。

### 模式 C：打包服务（高客单价）

**流程图：**
```
你 → 给客户部署 Hermes Web UI（只要电脑就能装）
         ↓
客户得到：
  ① 一个可访问的 Web 界面（浏览器打开就能用）
  ② 侧边栏自带「中转站」菜单
  ③ 你帮充值 ¥30 体验金
         ↓
收 ￥198 一次部署费
         ↓
后续客户自己在中转站充值 → 你从 apikey.fun 拿返佣
         ↓
持续收入
```

**定价参考：**
```
基础版：¥198/次  → 装 Web UI + 送体验金
标准版：¥398/次  → 装 Web UI + 送充值 + 教使用
```

**适用场景：** EPC 同行、台球俱乐部老板、不会技术的小白

## 六、重要认知（必读）

### 这不是传销，这是工具分发

你和 apikey.fun 的关系类似于：
- **App Store 开发者** 和 Apple（你推广，你拿提成）
- **知识付费推广员** 和课程平台（你推荐，你分佣）

不同之处在于：**你是先有工具，后有推广**。你的 Hermes Web UI 才是核心竞争力，中转站只是一个附加变现点。

### 为什么这件事能成

1. **零边际成本** — Web UI 装一个和装一百个没有区别
2. **自带转化** — 侧边栏的菜单替你推销
3. **刚需** — 每个人都想用上便宜的 AI
4. **你的圈子质量高** — EPC 老板们有消费能力
5. **小红书有现成受众** — 技能学院就是干这个的

### 风险提示

- **不要先大量囤货** — 先充 ¥200 测试，跑通再加大
- **不要承诺速度** — 中转站取决于上游供应商
- **不要碰违规内容** — 正常卖服务，不做黄牛、不刷单
- **客户付的钱别全花掉** — 留一部分作为备用buffer

## 七、FAQ（你可能会问的问题）

**Q：我完全不懂技术也能做吗？**
A：你已经在做了——你能用 Hermes，你就已经超越 95% 的人。

**Q：要投多少钱？**
A：前期 ¥0。注册免费。测试充值 ¥200 起步。

**Q：一个月能赚多少？**
A：项目初期不确定。但一个 ¥198 的部署套餐，每月 5 单就是 ¥1000。如果是返佣被动收入，取决于你的用户量。

**Q：这个平台靠谱吗？**
A：apikey.fun 基于 One API 开源项目，是行业标准方案。但**所有中转平台都存在被封/跑路风险**，所以不要囤大额，随用随充。

**Q：如果我客户多了，可以不依赖平台自己干吗？**
A：可以。有很多开源中转方案，但前期建议先用现成的，别人把基础设施、支付、客服都做好了，你只管卖。
