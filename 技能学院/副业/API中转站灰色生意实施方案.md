## 一、项目概述

搭建一个API中转代理服务，为国内用户提供海外大模型API（OpenAI、Claude、Gemini等）的转发访问。本质是 **"卖水"生意**——赚取流量差价。

### 核心公式
```
收入 = ∑(用户调用量 × 单价) - ∑(上游API成本 + 服务器成本)
利润率目标：50%-70%
```

---

## 二、技术架构

### 2.1 基础架构（最低成本版）

```
用户 → 域名/CDN → Nginx反代 → 鉴权/计费中间件 → 上游API(OpenAI/Claude等)
                                      │
                                      ▼
                                SQLite/MySQL (用量记录)
```

### 2.2 推荐技术栈

| 组件 | 推荐方案 | 用途 |
|---|---|---|
| **反代服务器** | Nginx + Lua | 请求转发 |
| **后端** | Node.js / Python FastAPI | 鉴权+计费 |
| **数据库** | SQLite（起步）/ MySQL（规模后） | 存储用户和用量 |
| **缓存** | Redis | 频率限制+热点缓存 |
| **部署** | 阿里云香港 / 腾讯云海外轻量 | 服务器位置 |
| **域名** | .xyz / .top 便宜域名 | 低调不备案 |

### 2.3 核心功能模块

1. **密钥管理**：生成用户的API Key，绑定额度
2. **请求转发**：标准OpenAI API兼容格式
3. **计费系统**：按token计费，支持预充值
4. **频率限制**：防止滥用
5. **用量监控**：给用户看仪表盘
6. **管理后台**：用户管理、价格设置、统计

## 三、部署步骤

### 第一阶段：搭建基础服务（1天）

```bash
# 1. 购买服务器（香港轻量，2核2G，约30元/月）
# 2. 安装基础环境
apt update && apt install -y nginx python3-pip redis-server

# 3. 部署中转核心（用开源项目 one-api 改）
git clone https://github.com/songquanpeng/one-api.git
cd one-api && docker-compose up -d
```

> 推荐直接用 **[one-api](https://github.com/songquanpeng/one-api)**（开源，Star 20k+）改改就能用，它已经包含：多模型接入、密钥管理、额度控制、Web管理面板。

### 第二阶段：对接上游渠道

| 上游 | 获取方式 | 成本 |
|---|---|---|
| **OpenAI** | 海外信用卡/虚拟卡注册 | GPT-4o: $2.5/1M input tokens |
| **Claude** | Anthropic API（需海外手机号） | Sonnet: $3/1M input tokens |
| **DeepSeek** | 国内直连，低延迟 | 极低，适合走量 |
| **中转上家** | 从其他中转站批发（省去自己出海） | 加价10-20% |

### 第三阶段：定价策略

| 模型 | 上游成本 | 建议售价 | 利润率 |
|---|---|---|---|
| GPT-4o | $2.5/1M input | ¥0.03/1K tokens | ~60% |
| GPT-4o-mini | $0.15/1M input | ¥0.005/1K tokens | ~70% |
| Claude Sonnet | $3/1M input | ¥0.035/1K tokens | ~60% |
| DeepSeek-V3 | ¥1/1M tokens | ¥0.003/1K tokens | ~50% |

**推荐策略**：主打低价引流（GPT-4o-mini接近成本），靠GPT-4o和Claude赚钱。

## 五、风险与规避

### ⚠️ 核心风险

| 风险 | 严重程度 | 应对 |
|---|---|---|
| **上游API封号** | 🔴 高 | 多账号轮换，每账号设日限额 |
| **被DDoS攻击** | 🟡 中 | Cloudflare WAF + 速率限制 |
| **法律风险** | 🟡 中 | 注册海外公司(Stripe Atlas)，国内低调运营 |
| **用户滥用** | 🟡 中 | IP+Key双限频，敏感内容过滤 |
| **支付跑路** | 🟢 低 | 预付费模式，不赊账 |

### 安全建议

1. **不要用国内服务器转发OpenAI** — 国内云厂商会拦截海外API
2. **域名用境外注册** — Namecheap/Namesilo，不用备案
3. **收款用USDT** — 匿名，减少监管风险
4. **隐私政策写清楚** — "仅做技术转发，不存储用户数据"
5. **不要接太高调** — 闷声发大财，别去大平台打广告
