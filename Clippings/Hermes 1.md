### 最该搞的：Auxiliary Models（辅助模型）

**简单说就是：给不同任务分配不同模型，别什么事都用同一个大模型。**

举个例子你就明白了——

你现在的情况：

> 你让 Hermes 做任何事，都是 DeepSeek （你的主力模型）在处理，哪怕只是：
> 
> - “压缩一下这段会话历史”
> - “帮我把这段文本翻译成中文”
> - “抓取这个网页的内容”
> 
> 这些 **不需要动脑子** 的活，也在用你花钱/花额度的主力模型。

配了辅助模型之后：

> **主力模型** （DeepSeek）只干需要思考的事：写代码、分析问题、决策 **辅助模型** （便宜的/本地的）干杂活：压缩会话、抓网页、翻译、格式化

效果：

- **省Token** ：辅助模型很便宜，甚至免费（比如本地的 qwen2.5）
- **速度快** ：小模型跑得快
- **主力模型不被打断** ：不会被琐事占用上下文

## 🔍 USB-Uncensored-LLM 研究结论

### 这是啥

| 🔌 **跨平台** | Shared 目录统一存模型，Win/Mac/Linux/Android 共享，不重复下载 |
| 🔥 **无审查模型** | Gemma 2 2B Abliterated（删了安全对齐）、Gemma 4 E4B Heretic（强制合规）、Qwen 3.5 9B Uncensored |

      ├── bin/         # 各平台 Ollama 引擎二进制
      ├── models/      # GGUF 权重大山

## 🔍 研究结果：Obsidian Web Clipper

**官网：** [obsidian.md/clipper](https://obsidian.md/clipper) **GitHub：** [obsidianmd/obsidian-clipper](https://github.com/obsidianmd/obsidian-clipper) ⭐4.3k，1001 commits **开发者：** Obsidian CEO kepano 亲自开发

### 它是什么？

官方出品的浏览器剪藏扩展，一键把网页内容剪到 Obsidian 本地库。支持 Chrome / Safari / Firefox / Edge / Brave / Arc / Vivaldi 所有主流浏览器。

### 核心功能

| 功能 | 说明 |
| --- | --- |
| **一键剪藏** | 点击扩展图标，自动提取文章正文 |
| **高亮模式** | 在网页上标注重点，只剪高亮内容 |
| **模板系统** | 可自定义保存格式，按网站自动匹配模板 |
| **元数据提取** | 自动抓取标题、作者、日期、 [schema.org](http://schema.org/) 结构化数据 |
| **热键** | `Cmd+Shift+O` 打开剪藏， `Alt+Shift+O` 快速剪藏 |
| **自然语言处理** | 可以用自然语言提问，提取页面指定内容 |
| **开源免费** | MIT 协议，无追踪 |

### 关于 YouTube 字幕提取

**Web Clipper 本身不直接提取 YouTube 字幕。** 它是网页剪藏工具，主要抓页面的正文内容。YouTube 的字幕（付费/隐藏字幕）是通过 YouTube 的 API 和 `?hl=zh-Hans` 等参数获取的，Web Clipper 的定位是剪藏可见页面的文本内容。

### 和你的体系对比

你现在已经有一套 YouTube 内容获取方案了—— `youtube-content` 技能（基于 youtube\_transcript\_api 提取字幕+AI 总结），已经集成到你本地的 Ollama + Hermes，能自动提取字幕、翻译、总结，直接存到 Obsidian。

### 我的建议

**这个插件还是值得装的** ，但不是为了 YouTube 字幕（你已经有更好的方案了），而是为了：

1. **日常网页剪藏** ——看到有价值的网页、文章、教程，一键存到 Obsidian 对应文件夹
2. **EPC 行业资料收集** ——看到建筑规范、政策文件、设计案例，直接剪到项目库
3. **小红书选题灵感** ——看到好的内容形式，直接剪到你的技能学院目录

**配合方案：**

- **YouTube 字幕/总结** → 继续用你现有的 `youtube-content` 技能（更专业）
- **普通网页/文章** → 用 Web Clipper 一键存（更快捷）
- **两者互补** ，不冲突
