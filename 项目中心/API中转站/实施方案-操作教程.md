### 2.4 令牌管理（零售端）

这是给你的"顾客"用的 API Key。

**创建令牌：**

1. 后台 -> 令牌 -> 添加令牌
2. 填入：
   - 名称：如 "客户-张三"
   - 用户：选择已有的用户账号（先创建用户）
   - 额度（可选）：限制总消费上限
   - 模型限制（可选）：只允许调用某些模型
3. 保存后复制 Token（仅显示一次）

**分享令牌 vs 独立令牌：**
- 分享令牌：多个 Token 共享同一个用户账号的额度
- 独立令牌：每个用户都有自己的用户账号和独立额度

**建议方案：**
- 免费用户：1个分享令牌（限额50元），所有体验用户共用
- 付费用户：每人独立用户+独立令牌

### 2.5 运营设置（定价核心）

这是你赚钱的关键——设置每个模型的零售倍率。

**位置：** 后台 -> 运营设置 -> 模型倍率

**推荐倍率：**

| 模型 | 进货价 | 倍率 | 零售价 | 毛利率 |
|------|--------|------|--------|--------|
| DeepSeek-V4-Flash | 1.0 | 3.0 | 3.0 | 67% |
| DeepSeek-V3.2 | 0.5 | 4.0 | 2.0 | 75% |
| DeepSeek-V4-Pro | 4.0 | 2.5 | 10.0 | 60% |
| Qwen3.5-235B | 2.0 | 3.0 | 6.0 | 67% |

**倍率逻辑：**
```
进货价 = 你从渠道拿到的价格
零售价 = 进货价 × 倍率
毛利 = 零售价 - 进货价
毛利率 = 毛利 ÷ 零售价 × 100%
```

## 第五部分：技术部署与运维

### 5.1 Docker 启动 One API

```bash
# 拉取镜像
docker pull justsong/one-api

# 启动（已有数据，升级版本）
docker stop one-api
docker rm one-api
docker run -d --name one-api --restart unless-stopped \
  -p 3001:3001 \
  -v /Users/mac/.hermes/one-api-data:/data \
  -e TZ=Asia/Shanghai \
  -e SESSION_SECRET=random_secret_string \
  justsong/one-api
```

**数据持久化路径：** `/Users/mac/.hermes/one-api-data/`
这里面是 SQLite 数据库文件，所有渠道、用户、Token、日志都在这里。定期备份这个目录！

### 5.2 Cloudflare Tunnel 对外暴露（已启动）

```bash
docker run -d --name cloudflared-tunnel \
  --restart unless-stopped \
  cloudflare/cloudflare:latest tunnel \
  --url http://host.docker.internal:3001

# 查看公网地址
docker logs cloudflared-tunnel 2>&1 | grep -o 'https://[a-z0-9.-]*\.trycloudflare\.com'
```

公网地址是一个 `https://xxx.trycloudflare.com` 格式的 URL，把这个地址发给客户即可。

### 5.3 日常运维命令

```bash
# 查看 One API 运行状态
docker ps | grep one-api

# 查看最近日志
docker logs one-api --tail 50

# 重启
docker restart one-api

# 备份数据
cp -r /Users/mac/.hermes/one-api-data \
```

## 第九部分：常见问题

### Q: 用户反映速度慢怎么办？

A: 看你用的是哪个渠道。SiliconFlow 国内直连速度最快。检查 One API 后台日志，看请求走了哪个渠道，考虑换渠道或添加更多渠道做负载均衡。

### Q: 用户 API Key 被曝光被盗用怎么办？

A: 在 One API 后台直接禁用该 Token，然后给用户重新生成一个。如果你的主管理 Token 泄露，立刻去后台->设置修改 Token。

### Q: 渠道的 API Key 失效了怎么办？

A: 去该渠道网站重新生成 API Key，然后在 One API 后台更新对应渠道的信息，测试连通后保存。

### Q: 免费用户用超了 50 元额度？

A: 在"令牌"页面修改额度限制，或者直接禁用该 Token。免费用户不需要太客气，超过额度就停。

### Q: 怎么知道哪个渠道最便宜？

A: 跑 `python3 ~/.hermes/scripts/profit-tracker.py`，看"按渠道分析"部分，比较各渠道的成本。

### Q: One API 挂了怎么恢复？

A:
```bash
docker restart one-api
# 如果不行
docker stop one-api && docker rm one-api
docker run -d --name one-api --restart unless-stopped \
  -p 3001:3001 \
  -v /Users/mac/.hermes/one-api-data:/data \
  -e TZ=Asia/Shanghai \
  justsong/one-api
```

### Q: 怎么给用户显示使用统计？

A: One API 自带的日志页面可以查看每个用户的调用情况。后台 -> 日志，筛选用户名即可。用户自己也可以访问 `/api/log/self` 查看自己的使用记录（需要他们的 Token）。

### Q: 怎么防止单一渠道出问题影响全部用户？

A: 同一个模型配置多个渠道。One API 支持"渠道分组"，把多个同类型渠道分到一组，自动失败切换。
