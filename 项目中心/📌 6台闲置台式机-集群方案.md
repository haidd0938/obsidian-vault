# 6 台闲置台式机 · 四合一集群方案

> 老板有 6 台闲置老台式机，想利用起来干四件事：
> 1. 🔥 本地 AI 推理集群（跑大模型）
> 2. 🎬 视频渲染集群（鑫球汇短视频）
> 3. 🕷️ 爬虫集群（Scrapling 大规模采集）
> 4. 🖥️ NAS + HPC 计算节点
>
> **原则**：一套系统四件事都能干，不搞四套独立方案。

---

## 🏗️ 一、推荐架构：Docker Swarm + 角色分工

```
                  ┌─────────────────────┐
                  │   Manager Node #1    │
                  │   (Ollama + NAS)     │
                  │   SSD: 系统+模型     │
                  │   HDD: 共享存储      │
                  └────────┬────────────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
   ┌──────┴──────┐  ┌─────┴──────┐  ┌─────┴──────┐
   │ Worker #2   │  │ Worker #3  │  │ Worker #4  │
   │ Ollama推理   │  │ Ollama推理  │  │ Ollama推理  │
   │ + 爬虫节点   │  │ + FFmpeg   │  │ + 爬虫节点   │
   └─────────────┘  └────────────┘  └────────────┘
   ┌─────────────┐  ┌────────────┐
   │ Worker #5   │  │ Worker #6  │
   │ 视频渲染主力  │  │ HPC计算     │
   │ (FFmpeg负载)  │  │ (Python)   │
   └─────────────┘  └────────────┘
```

### 角色分配逻辑

| 编号 | 角色 | 职责 | 推荐硬件 |
|------|------|------|---------|
| #1 | **Manager** | 统筹调度、NAS存储、Ollama服务 | 配置最高那台（SSD系统盘 + 大容量HDD） |
| #2 | **推理+爬虫** | Ollama推理节点 + 爬虫任务 | CPU多核优先 |
| #3 | **推理+渲染** | Ollama推理 + 视频渲染副节点 | CPU多核优先 |
| #4 | **推理+爬虫** | 同上#2 | CPU多核 |
| #5 | **渲染主力** | FFmpeg批量渲染，夜间开干 | CPU多核 |
| #6 | **HPC计算** | Python科学计算/数据处理 | CPU多核 |

> ⚠️ **关键前提**：老台式机大概率只有 CPU（无独显或亮机卡）。所有推理/渲染/计算都跑在 CPU 上。这正好——老机器的 CPU（尤其 E5 这种洋垃圾级）多核干活很猛，反而比 GPU 推理更省心（不用配 CUDA）。

## 🔥 方案一：本地 AI 推理集群（Ollama 分布式）

### 怎么在多台机器上跑同一个模型？

Ollama 原生不支持跨节点分布式推理，但有三种实用方案：

#### 🥇 方案A：Ollama + Nginx 负载均衡（推荐）

```
                    ┌──────────┐
                    │  Nginx   │ (Manager #1)
                    │ :11434   │
                    └────┬─────┘
                ┌────────┼────────┐
                │        │        │
         ┌──────┴─┐ ┌───┴────┐ ┌─┴──────┐
         │Ollama#1│ │Ollama#2│ │Ollama#3│
         │(#1)    │ │(#2)    │ │(#3)    │
         └────────┘ └────────┘ └────────┘
```

- Nginx 做反向代理，轮询分发给各节点的 Ollama
- 你的 MacBook 上的 Hermes 把 Ollama endpoint 指向 Nginx 地址
- **效果**：多个模型实例并行，高并发场景吞吐量翻倍

```
# 在 Manager #1 上配置 Nginx
upstream ollama_cluster {
    server 192.168.1.101:11434;  # Node #1
    server 192.168.1.102:11434;  # Node #2
    server 192.168.1.103:11434;  # Node #3
    server 192.168.1.104:11434;  # Node #4
}
server {
    listen 11434;
    location / { proxy_pass http://ollama_cluster; }
}
```

#### 🥈 方案B：分模型+分布式推理

| 机器 | 跑的模型 | 用途 |
|------|---------|------|
| MacBook Pro (你主力机) | DeepSeek / API | 主力工作推理 |
| #1 Manager | Ollama: gemma4 (15B) | 本地高精度推理 |
| #2 + #3 | Ollama: qwen2.5 (7B) | 日常轻量推理 |
| #4 | 跑小模型 (llama3.2 3B) | 实时/低延迟任务 |
| #5 + #6 | 纯跑渲染/计算 | 不分担推理负载 |

#### 🥉 方案C：llama.cpp + rpc 分布式推理

llama.cpp 有 `--rpc` 参数，可以把推理层拆到多台机器：
- Manager 加载模型 → 各 Worker 做部分计算 → 合并结果
- **适合超大模型**（比如 70B 参数，单机内存不足时）
- 老机子 32GB 内存的，6台合一起大概能跑 70B q4 量化模型

## 🎬 方案二：视频渲染集群（FFmpeg + Docker）

### 架构：Tasks 分发队列

```
Manager #1  →  Redis队列(任务池)  →  Worker们拉取任务
                                        │
                        ┌───┬───┬───┬───┘
                        │   │   │   │
                       #3  #5  #6  其他
```

### 工作流

```
你的 MacBook (发布任务)
       │
       ▼
Manager #1 (分发到 Redis)
       │
       ▼
          ┌─────────────┐
          │   Redis     │  →  Worker #3: 取任务1 (片A 0-30s)
          │ 任务队列     │  →  Worker #5: 取任务2 (片B 30-60s)
          └─────────────┘  →  Worker #6: 取任务3 (片C 60-90s)
                ↓
          Manager #1 合并 → 返回成品视频
```

### 适合的场景

- 鑫球汇短视频每天 4 条 × 不同路线 → 每条拆成 8 段 60s → 总 32 个片段 → 集群并行渲染 16 分钟活变 2 分钟
- 批量压缩/转码 → 把积累的素材库统一转格式
- 批量添加水印/字幕 → 一个命令跑遍所有文件

### 开销

- 电费：6 台满载约 1500-2400W/h，晚上 0.5 元/度 ≈ 1-2 元/小时
- 软件：全免费（FFmpeg + Docker + Redis）

## 🕷️ 方案三：爬虫集群（Scrapling + 代理轮换）

### 架构

```
MacBook (你的调度中心)
    │
    ├── 发爬虫任务到 Redis
    │
    └── 各 Worker 取任务，每个 Worker 独立 IP
        ├── #2: 爬大众点评（代理 A）
        ├── #4: 爬招标信息（代理 B）
        └── #6: 爬备选源（代理 C）
```

### Scrapling 部署在每台 Worker 上

```bash
# 每台机器都装
pip install scrapling[all]
scrapling install  # 安装 Playwright 浏览器
```

### 核心优势

| 特性 | 单机 | 6台集群 |
|------|------|--------|
| 并发请求 | 8-16 | 48-96 |
| IP指纹 | 1个 | 6个不同（代理池更大） |
| 被封概率 | 高（被识别为同一来源） | 低（6个不同环境） |
| 日采集量 | ~1万条 | ~5-8万条 |
| 代理成本 | 相同（每台可用不同代理出口） | 更划算（代理复用度低） |

### 反爬策略增强

- 每台机器装不同的浏览器版本（Chrome 120/121/122 等）
- 每台用不同的 User-Agent + TLS 指纹
- 每台连不同的代理 → 6 个独立 IPv4 出口

## 🖥️ 方案四：NAS + HPC 计算节点

### NAS (Manager #1)

```
Manager #1
├── SSD(256G) → 系统 + Docker + Ollama模型
└── HDD(1T+)  → NFS/Samba 共享给所有节点
                 ├── 模型存储（ollama模型文件夹）
                 ├── 视频素材库（原始素材）
                 ├── 视频输出目录（成品）
                 ├── 爬虫数据目录（采集结果）
                 └── 代码仓库（各节点同步）
```

挂载方式：

```bash
# 所有 Worker 挂载 Manager 共享目录
# Manager 上
sudo apt install nfs-kernel-server
echo "/data *(rw,sync,no_subtree_check)" >> /etc/exports

# Worker 上
sudo mount -t nfs manager-ip:/data /mnt/data
```

### HPC 计算（Worker #6 + 其他空闲节点）

Python 分布式计算框架选择：

| 框架 | 上手 | 适用场景 |
|------|------|---------|
| **Ray** | ⭐⭐⭐⭐⭐ | 通用分布式计算，Python 生态兼容最好 |
| **Dask** | ⭐⭐⭐⭐ | 数据分析/科学计算首选 |
| **Celery** | ⭐⭐⭐⭐ | 任务队列，爬虫/视频渲染也适用 |

推荐 **Ray**——它是现在最火的分布式框架，开箱即用：

```python
# Manager 启动
ray start --head --port=6379

# 每个 Worker 加入
ray start --address=manager-ip:6379

# Python 代码里用（来自你的 MacBook 或任意节点）
import ray
ray.init(address="manager-ip:6379")

@ray.remote
def batch_process(data):
    # 这个函数会在集群里任意节点执行
    return heavy_computation(data)

# 自动并行分发
results = ray.get([batch_process.remote(d) for d in big_dataset])
```

## 📋 部署步骤

### 第一步：摸底（先搞清楚你6台什么配置）

需要你回答的问题：

```
每台机器的：
1. CPU型号（Intel E5? i5? i7? 第几代？）
2. 内存多大？（8G? 16G? 32G?）
3. 硬盘多大？SSD还是HDD？
4. 有没有独显？
5. 有没有装过系统？Windows还是Linux？
```

或者可以：**你先挑一台插上显示器开机，跑 `systeminfo` 或 `lscpu` 看看配置**，一台的配置基本能代表其他几台。

### 第二步：确定 Manager 节点

- 挑配置最高的一台做 Manager
- 装 Ubuntu Server 24.04 LTS
- 系统装 SSD（如果没有SSD，买个120G SSD ≈ 50块钱）
- 大容量 HDD 做 NAS 共享盘

### 第三步：批量部署 Worker

**方案A：PXE 网络批量装系统（省U盘）**
- Manager 做 DHCP + TFTP 服务器
- 其他 5 台网卡启动，自动装 Ubuntu
- *一次装 5 台，不用插U盘*

**方案B：传统U盘安装**
- 一个 8G U盘，装 Ubuntu Server 24.04
- 每台插一次，15分钟/台
- 5 台 ≈ 1.5 小时

**方案C：保持 Windows，用 WSL2**
- 如果机器上有 Windows 系统，装 WSL2 + Docker Desktop
- 不用重装系统，最快上手

### 第四步：安装核心软件

```bash
# 全 6 台都跑一遍
sudo apt update && sudo apt install -y docker.io docker-compose
sudo systemctl enable --now docker

# Manager 初始化 Swarm
docker swarm init --advertise-addr manager-ip

# Worker 加入 Swarm
docker swarm join --token xxx manager-ip:2377

# 安装 Ollama（全 6 台）
curl -fsSL https://ollama.com/install.sh | sh

# 安装 Python 全家桶（全 6 台）
sudo apt install -y python3 python3-pip python3-venv ffmpeg
pip install scrapling[all] requests beautifulsoup4

# Manager 额外装
sudo apt install -y nfs-kernel-server redis nginx
pip install ray[default]

# Worker 额外装
pip install ray[default]
```

### 第五步：启动各服务

```bash
# 1. Manager 起 NAS（NFS/Samba）
sudo systemctl enable --now nfs-kernel-server

# 2. Manager 起 Redis（任务队列）
docker run -d --name redis -p 6379:6379 redis

# 3. Manager 起 Ollama + 配置 Nginx 负载均衡

# 4. 各 Worker 启动 Ollama
ollama serve &

# 5. Manager 起 Ray 头节点
ray start --head --port=6380

# 6. Worker 加入 Ray
ray start --address=manager-ip:6380
```

### 第六步：接入你的 MacBook

```python
# 从你的 MacBook 上直接调用集群的 Ollama
import requests
response = requests.post("http://manager-ip:11434/api/chat", json={
    "model": "qwen2.5",
    "messages": [{"role": "user", "content": "你好"}]
})

# 或者改 Hermes 配置指向集群
# ~/.hermes/config.yaml 里 ollama 的 host 改成 manager-ip
```

## ⚠️ 注意事项

### 散热和噪音
- 6台老机器一起开，**噪音不小**（风扇呼呼的）
- **建议放车库/阳台/储藏室**，别放卧室和书房
- 夏天建议只开2-3台，或开空调放

### 网络
- 必须**千兆有线局域网**，WiFi 撑不住多台机器的大文件传输
- 建议配个千兆交换机（50块钱），再接路由器

### 稳定性
- 老机器硬盘容易挂 → 重要数据不要只放一台
- **Manager 的 NAS 盘建议监控健康状态**：`sudo smartctl -a /dev/sda`

### 安全
- 内网集群，**不要直接暴露到公网**
- 如果要从外网访问 → 走你的 MacBook 做跳板机（SSH tunnel）

## 🚀 建议优先级

你的情况我建议这么来：

```
第一周：摸底配置 → Manager 装系统 → 测试 1 台
第二周：扩到 3 台 → NAS + Ollama 负载均衡跑起来
第三周：扩到 6 台 → 加入爬虫 + 视频渲染
按需：接入 Ray 做 HPC 计算
```

**先搞定 1 台，别一次性上 6 台**——先看看噪音、温度、实际收益能不能接受，再扩张。
